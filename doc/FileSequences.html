Overview

A file sequence is a group of files that represent individual frames of a film or animation sequence. DJV will recognize these groups of files and display them as a sequence in the user interface:

render.1.exr

For a file to be recognized as part of a sequence it must have a frame number at the end of it's name and the same file extension. Here are some examples of different naming conventions for file sequences:

Formatting

Formatting changes how the file sequences are displayed:

The "sparse" setting will show the exact frames in the sequence including any gaps where there may be missing frames. The "range" setting will show a continous sequence even if there are missing frames. The formatting can be set either in the file browser, the preferences, or the command-line.

Auto Sequencing

Auto sequencing will try and automatically determine the file sequence from a single file. This allows you to drag and drop a file onto DJV and have the associated file sequence opened automatically. This option may be turned on or off in the preferences or the command-line.

Maximum Size

The maximum size limits how many frames are in a file sequence. This value may be set in the preferences or the command-line.

Negative Numbers

By default negative numbers are not enabled for file sequences since they can cause issues with some naming conventions. This option may be enabled in the preferences or the command-line.
