Requirements

Build tools:

  • C++11 compiler
  • CMake
  • git

Libraries:

  • OpenGL 3.3
  • GLM 0.9.9
  • IlmBase 2.3
  • Qt 5.6.3

Optional libraries:

  • libjpeg v9b
  • libpng 1.6
  • libtiff 4.0
  • OpenEXR 2.3
  • FFmpeg 4.0

Third Party Libraries

DJV relies on a number of third party libraries that can either be installed with your system or built from source code. The following steps describe how to build the third party libraries from source code.

First choose a working directory for building the source code. In the following steps we will refer to this directory as $BUILD_DIR.

Clone the DJV third party source code repository:

> git clone git://git.code.sf.net/p/djv/git-third-party djv-git-third-party

This source code repository contains the third party libraries as well as minor modifications necessary for building on the various platforms that DJV supports.

Create a sub-directory for building the third party libraries and start the build:

> mkdir djv-git-third-party-Debug
> cd djv-git-third-party-Debug
> cmake ../djv-git-third-party \
    -DCMAKE_BUILD_TYPE=Debug \
    -DCMAKE_PREFIX_PATH=$BUILD_DIR/djv-install-Debug \
    -DCMAKE_INSTALL_PREFIX=$BUILD_DIR/djv-install-Debug \
    -DCMAKE_FIND_FRAMEWORK="LAST"
> make
> cd ..

After the build completes add the install path to your LD_LIBRARY_PATH:

> export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:$BUILD_DIR/djv-install-Debug/lib

Building the Source Code

First choose a working directory for building the source code. In the following steps we will refer to this directory as $BUILD_DIR.

Clone the DJV source code repository:

> git clone git://git.code.sf.net/p/djv/git djv-git

Create a sub-directory for building the DJV software and start the build.

Building with system libraries:

> mkdir djv-git-Debug
> cd djv-git-Debug
> cmake ../djv-git \
    -DCMAKE_BUILD_TYPE=Debug
> make

Building with local libraries:

> mkdir djv-git-Debug
> cd djv-git-Debug
> cmake ../djv-git \
    -DCMAKE_BUILD_TYPE=Debug \
    -DCMAKE_PREFIX_PATH=$BUILD_DIR/djv-install-Debug \
    -DCMAKE_INSTALL_PREFIX=$BUILD_DIR/djv-install-Debug
> make

If you are compiling for OS X you should also set the CMAKE_FIND_FRAMEWORK variable:

> mkdir djv-git-Debug
> cd djv-git-Debug
> cmake ../djv-git \
    -DCMAKE_BUILD_TYPE=Debug \
    -DCMAKE_PREFIX_PATH=$BUILD_DIR/djv-install-Debug \
    -DCMAKE_INSTALL_PREFIX=$BUILD_DIR/djv-install-Debug \
    -DCMAKE_FIND_FRAMEWORK="LAST"
> make

After the build completes try running the confidence tests:

> make test

You can also try starting the djv_view application:

> ./build/bin/djv_view

Packaging Releases

Set the new version in the top-level "CMakeLists.txt" and "Doxyfile" files.

Update the download information in "doc/Download.html".

Tag the git repository:

> git tag -a 1.2.2 -m "Version 1.2.2"
> git push origin 1.2.2

Build the package:

> cmake ../djv-git \
    -DCPACK_GENERATOR=RPM
> make package
